# Skill Governance

Skill 是能力扩展，不是默认安全资产。

## 流程

1. **发现**：从已知 skill 源筛选候选
2. **审查**：读文件、看脚本、查高风险模式
3. **建议**：说明适用场景、收益、风险、维护状态
4. **确认**：得到明确确认后再安装
5. **回验**：安装后验证可见、可用、无副作用

## 必查项

- 是否存在 `curl` 到未知 URL
- 是否存在 `eval`、可疑 `base64`、隐藏下载行为
- 是否强依赖高权限命令
- 是否修改工作区外关键路径

## 红线

- 未经确认，不自动安装
- HIGH 风险及以上默认拒绝
- 不把 skill 当黑盒；至少看入口文件和关键脚本
