# Spec Template

`aic-planner` 统一使用以下结构输出需求规格。

```md
# Spec

任务ID: <task_id>

## 背景
<为什么要做>

## 目标
<本次必须达成什么>

## 不做项
<本次明确不做什么>

## 用户路径 / 使用场景
- 场景 1
- 场景 2

## 交付物
- 交付物 1
- 交付物 2

## 验收标准
- [ ] 标准 1
- [ ] 标准 2

## 技术约束
- 约束 1
- 约束 2

## 风险
- 风险 1
- 风险 2

## 实施建议
- 第一步
- 第二步
- 第三步
```

## 红线

- 不写“尽量做好”这类空话
- 不允许没有不做项
- 不允许没有验收标准
