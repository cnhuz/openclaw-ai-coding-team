# Release Gate

`aic-releaser` 在任何正式上线前都必须检查以下门禁。

## 发布前门禁

- 需求范围已冻结
- 关键实现已完成
- 测试结果已出
- 必要环境变量已确认
- 数据迁移脚本已确认
- 回滚方案已确认
- 上线窗口已明确
- 烟测清单已准备

## 强制检查项

### 1. 实现

- 本次改了什么
- 改动是否超出原计划
- 是否影响历史兼容性

### 2. 验证

- 关键路径是否测试通过
- 是否有未解决 blocker
- 是否存在高风险但未验证项

### 3. 部署

- 部署命令
- 依赖服务变更
- 配置变更
- 数据库变更

### 4. 回滚

- 回滚命令
- 回滚前提
- 回滚后检查项

### 5. 观察

- 发布后 10 分钟观察什么
- 发布后 1 小时观察什么
- 需要谁值守

## 禁止直接上线的情况

- `aic-reviewer` 未批准
- `aic-tester` 报告中仍有 blocker
- 没有回滚方案
- 不知道改动影响面
- 部署会外发 / 删除 / 重置关键数据但未取得明确批准
