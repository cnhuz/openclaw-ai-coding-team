# Opportunity Card

`aic-researcher` 在发现新方向或新需求时，统一使用以下结构。

```md
# Opportunity Card

## 标题
<一句话描述机会>

## 面向谁
<用户 / 场景 / 角色>

## 机会来源
- 用户反馈：
- 竞品变化：
- 社区讨论：
- 内部观察：

## 证据
- 证据 1
- 证据 2
- 证据 3

## 为什么值得做
- 对用户的价值
- 对产品的价值
- 对增长 / 收入 / 留存的价值

## 商业模式
- 低价订阅 / 一次性付费 / 广告 / 开源转化 / 模板销售 / 其它

## 付费假设
- 谁会掏钱
- 为什么现在愿意掏钱
- 为什么不会只停留在“看起来有需求”

## 分发路径
- SEO / 社区传播 / GitHub / Product Hunt / 社媒 / 目录 / 其它

## 单位经济性
- token / infra / 维护成本是否可控
- 低价产品是否仍有空间

## 自动化适配
- 这个机会是否适合 agent 团队自动化构建、低维护运营
- 是否会拖入重服务、重定制、重人工支持

## 风险
- 技术风险
- 需求风险
- 交付风险

## 止损条件
- 满足哪些信号时应停止投入或降级为观察

## 成功指标
- 收入 / 流量 / 激活 / 留存 / 复购 / 转化 等最小验证指标

## 建议动作
- 建议继续研究 / 立项 / 暂缓

## 建议优先级
<P0 / P1 / P2>
```

## 使用规则

- 没有证据，不要立卡
- 机会卡是输入，不是承诺
- 真正实施前仍需 `aic-planner` 收敛范围
- 机会卡不仅要回答“值不值得做”，还要回答“是否有机会养活团队”
